**Note on the two checks that failed on the first pass:** the automated script's assertions were
wrong, not the app.
- *Workers page:* the script expected to see a running worker. This test registers a brand-new
  project, and no worker process was ever started for it — so the correct behavior is exactly what
  rendered: the "no workers yet" guide with that project's real UUID pre-filled into a copy-pasteable
  `WORKER_PROJECT_ID=... python -m app.workers.main` command.
- *Audit log:* the script searched for lowercase `"project.created"`; the UI renders action names
  uppercase via CSS (`PROJECT.CREATED`). The entries were there and correct — the search string was
  case-sensitive when it shouldn't have been.

Both are recorded here as a check on the *checker*, not swept under the rug.
